#### "What language should I start with?"
If You've got any experience with programming or ever wanted to start, You asked Yourself that question
many times. And the answer You've probably heard was:

#### "Start with C/C++ and then move to more advanced languages like Java"

I know You did. I hear that all the time, and I went that way, and I almost gave up programming because of that.
Let's make it clear. C++ is a horribly hard language. It's concepts are complicated to every day proffesional programmers,
not to mention newcomers. With all it's memory management, post-experimental Object Oriented features and pointer madness. 
Telling someone that he has to learn memory management to learn programming is like saying You need to understand gas
refinement to learn driving.

To really choose the best language to start learning we need to define what features define good learning material.

1. **It's fun!** - I think I don't need to explain that. Learning a new thing cannot be effective if You don't feel entertained by it. You just can't. Period
2. **Start general, continue specific** - You can't expect somebody to appreciate details about a problem that he/she doesn't get in the first place.
3. **It's based on real world problems** - *"Implement Euclidean Algorithm"* - JUST NO! For gods' sake
4. **All of the errors are easily understandable** - There is an error in line 15 because line 14 misses a semicolon. Not really what we want to deal with
5. **No copy pasting** - The code has to be short, easy to understand, and self-explanatory
6. **Allows quick experiments** - Coder **must** be able to input short expressions and check their results in no time.
